## sorted set을 이용한 실시간 리더보드

리더보드란 경쟁자의 순위와 현재 점수를 보여주는 순위표를 의미한다. 주로 게임에서 스코어로 정렬돼 상위 경쟁자의 순위를 보여주는 용도로 쓰지만, 게임 외의 서비스에서도 여러 데이터들을 게임화해 리더보드로 나타내는 방식을 자주 사용하고 있다.

절대적 리더보드는 서비스의 모든 유저를 정렬시켜 상위권의 목록만 표시.

상대적 리더보드는 사용자마다 다른 데이터를 보여준다. 사용자의 스코어를 기반으로 다른 사용자와 비교해 순위를 결정하는 형태의 리더보드다.

```
127.0.0.1:6379> ZADD daily-score:220817 28 player:286
(integer) 1
127.0.0.1:6379> ZADD daily-score:220817 400 plaher:234
(integer) 1
127.0.0.1:6379> ZADD daily-score:220817 45 player:101
(integer) 1
127.0.0.1:6379> ZADD daily-score:220817 357 player:24
(integer) 1
127.0.0.1:6379> ZADD daily-score:220817 199 player:143
(integer) 1
```

sorted set에는 데이터가 스코어 순으로 정렬돼 저장된다.

```
127.0.0.1:6379> ZRANGE daily-score:220817 0 -1 withscores
 1) "player:286"
 2) "28"
 3) "player:101"
 4) "45"
 5) "player:143"
 6) "199"
 7) "player:24"
 8) "357"
 9) "plaher:234"
10) "400"
```

데이터 업데이트

만약 player:286이 게임을 해서 데이터를 업데이트한다면

```

(integer) 0
127.0.0.1:6379> ZRANGE daily-score:220817 0 -1 withscores

 3) "player:143"
```

## 래다스에서의 다양한 카운팅 방법

애플리케이션에서 데이터의 개수를 세는 카운팅 작업은 자주 발생한다.

### 좋아요 처리하기

실시간 트래픽이 굉장히 많은 사이트라면 하나의 뉴스 댓글에 좋아요가 눌리는 일은 1초에 몇 만개 이상 발생할 수 있으며, 디비에 개수를 증가시키는 일은 디비에 직접적인 영향을 미칠 수 있다.

중복 처리하지 않고 어떤 유저가 어떤 댓글에 좋아요를 눌렀는지의 데이터 또한 처리할 수 있어야 한다.

이를 구현하는 데 레디스의 set을 간단히 활용할 수 있다.

### DAU 구하기

DAU (Daily Active User)는 하루 동안 서비스에 방문한 사용자의 수를 의미한다. 하루에 여러번 방문했다 하더라도 한 번으로 카운팅되는 값으로, 실제 서비스를 이용한 사용자의 유니크한 수를 파악할 수 있는 지표다.

하루 동안 방문했던 유저 ID를 set에 저장하는 방법도 고려할 수 있다. 하지만 만약 하루 1000만 명 이상의 유저가 방문하면 큰 서비스라면 이럴 경우 하나의 키 안에 너무 많은 아이템이 저장된다.

레디스의 비트맵을 이용하면 메모리를 효율적으로 줄이면서 실시간으로 서비스의 DAU를 확인할 수 있다.
