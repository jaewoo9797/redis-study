# 복제

## 레디스에서의 복제 구조

### 복제 구조 구성하기

레디스에서 복제를 사용하는 방법은 굉장히 간단하다.

```
REPLICAOF <mater-ip> <mater-port>
```

레디스의 데이터를 업데이트하는 모든 커맨드는 마스터 노드에서 실행되기 때문에 서비스 애플리케이션은 A를 바라보아야 한다.

### 패스워드 설정

## 복제 메커니즘

레디스에서 복제는 자동으로 이뤄지며 사용자의 개입이 필요치 않다.

1. REPLICAOF 커맨드로 복제 연결을 시도한다.
2. 마스터 노드에서는 fork로 자식 프로세스를 새로 만든 뒤 RDB 스냅숏을 생성한다.
3. 2번 과정 동안 마스터 노드에서 수행한 모든 데이터셋 변경 작업은 레디스 프로토콜 형태로 마스터의 복제 버퍼에 저장된다.
4. RDB 파일이 생성 완료되면 파일은 복제본 노드로 복사된다.
5. 복제본에 저장됐던 모든 내용을 모두 삭제한 뒤 RDB 파일을 이용해 데이터를 로딩한다.
6. 복제 과정 동안 버퍼링된 복제 버퍼의 데이터를 복제본으로 전달해 수행시킨다.

버전 7 이후부터 디스크를 사용하지 않느 방식.
